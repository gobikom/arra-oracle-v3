## Summary

Closed the 3 remaining architectural findings from the PR #716 multi-agent review of the OAuth 2.1 provider:

- **#17.4** — `redirect_uri` scheme validation (allowlist: https://, http://localhost, http://127.0.0.1, ::1) enforced at BOTH `registerClient` and `authorize` (defense in depth).
- **#17.5** — RFC 7009 client authentication on `/revoke` with HTTP Basic (preferred) or form-body credentials (mutually exclusive per RFC 6749 §2.3.1), HMAC-normalised timing-safe client_secret compare, and token-client binding enforcement (client A cannot revoke client B's tokens, no existence leak).
- **#17.7** — Hash-before-lookup refactor: `state.tokens` is now keyed by `sha256hex(token)`. The dict-key timing side-channel is closed at issue, verify, and revoke paths. Legacy state files are migrated in place via a one-shot `migrateTokenKeys` pass in `loadState`, with a `tokenKeyVersion: 'sha256'` marker for idempotency. Migration is eager-persisted in the constructor so a crash between load and the next write does not retrigger.

## Assessment vs Reality

| Metric | Predicted | Actual | Reasoning |
|--------|-----------|--------|-----------|
| Complexity | HIGH | HIGH | All 3 findings required touching provider.ts core paths + routes wiring. |
| Confidence | 9/10 | 9/10 | Plan was solid; only deviation was consolidating the integration tests into the existing unit test files via `app.request()` instead of spawning a new HTTP server (faster, no new flake surface). |

**Deviation**: The plan envisioned extending `src/integration/oauth-hardening.test.ts` with HTTP-spawned integration tests (ports 47783-47785). In implementation I used `hono`'s `app.request()` interface in the existing unit test files (`routes-revoke.test.ts`), which exercises the SAME HTTP contract without spawning a subprocess. This gives the same coverage in ~80ms vs ~15s per test file and avoids the pre-existing `ORACLE_API_TOKEN`-required flakiness in the integration harness. Net: all 3 findings have comprehensive contract coverage.

**Second deviation**: IPv6 loopback hostname handling — Bun's WHATWG URL returns `hostname` with brackets (`[::1]`) whereas Node strips them. Accepted both shapes in `validateRedirectUri` after the first unit test run surfaced the divergence.
